Node A is the **self-evolving AI brain** of the home lab — a complete inference stack centred on the AMD RX 7900 XT.

| Item | Value |
|------|-------|
| Role | Brain — heavy inference, RAG, web search, code execution |
| IP | `192.168.1.9` |
| CPU | Intel Core i9-265F |
| RAM | 128 GB DDR5 |
| GPU | **AMD Radeon RX 7900 XT (20 GB VRAM)** |
| GPU arch | RDNA 3 / gfx1100 |
| Base model | Dolphin 2.9.3 Llama 3.1 8B AWQ (uncensored) |

## 1 — Host Prerequisites

### 1.1 Operating System

**Fedora 44** is the primary supported platform. Ubuntu 22.04/24.04 notes are retained as `[Ubuntu]` callouts where the commands differ.
The RX 7900 XT requires **ROCm 6.x** for GPU-accelerated inference.

### 1.2 ROCm (AMD GPU driver stack)

**Fedora 44 (primary):**
```bash
sudo tee /etc/yum.repos.d/rocm.repo > /dev/null <<'REPO'
[ROCm]
name=ROCm
baseurl=https://repo.radeon.com/rocm/rhel9/latest/main
enabled=1
gpgcheck=1
gpgkey=https://repo.radeon.com/rocm/rocm.gpg.key
REPO

sudo dnf install -y rocm-hip-sdk rocm-opencl-sdk rocminfo rocm-smi-lib
```

> **[Ubuntu 22.04 / 24.04]**
> ```bash
> wget -q -O /tmp/amdgpu-install.deb \
>   "https://repo.radeon.com/amdgpu-install/6.1.3/ubuntu/jammy/amdgpu-install_6.1.60103-1_all.deb"
> sudo apt-get install -y /tmp/amdgpu-install.deb
> sudo amdgpu-install --usecase=rocm --no-dkms -y
> ```

### 1.3 Add user to GPU groups

```bash
sudo usermod -aG render,video "$USER"
# Log out and back in (or: newgrp render)
```

### 1.4 Verify ROCm sees the GPU

```bash
/opt/rocm/bin/rocminfo | grep -A5 "Marketing Name"
# Expected: "Radeon RX 7900 XT"

ls -la /dev/kfd /dev/dri/render*
# /dev/kfd must exist; /dev/dri/renderD128 (or similar) must exist

rocm-smi
# Should list the GPU with VRAM and utilisation
```

### 1.5 Docker

**Fedora 44 (primary):**
```bash
sudo dnf install -y dnf-plugins-core
sudo dnf config-manager --add-repo \
  https://download.docker.com/linux/fedora/docker-ce.repo
sudo dnf install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
sudo systemctl enable --now docker
sudo usermod -aG docker "$USER"
newgrp docker
```

> **[Ubuntu]** Follow the official Docker install guide: https://docs.docker.com/engine/install/ubuntu/

## 2 — Quick Deployment (Recommended)

### 2.1 Configure environment

```bash
cd ~/homelab/node-a-vllm
cp .env.example .env
nano .env      # set HUGGING_FACE_HUB_TOKEN at minimum
```

Key `.env` variables:
```bash
# Required — HuggingFace token to download the Dolphin model weights
HUGGING_FACE_HUB_TOKEN=hf_your_token_here

# Auto-generated by setup.sh if not set manually
WEBUI_SECRET_KEY=<random 32-char hex>
SEARXNG_SECRET=<random 32-char hex>
JUPYTER_TOKEN=<random 16-char hex>

# Optional — leave blank for unauthenticated Qdrant
QDRANT_API_KEY=
```

### 2.2 Run the setup script

```bash
# Full setup: ROCm check, env prep, image pull, deploy
./node-a-vllm/setup.sh

# Or from the repo root via scripts/setup-node-a.sh
./scripts/setup-node-a.sh

# Install ROCm only, skip deploy
./scripts/setup-node-a.sh --no-deploy

# Check status of all services
./scripts/setup-node-a.sh --status
```

### 2.3 Verify

```bash
# vLLM health (returns 200 when model is loaded — may take 3-5 min on first start)
curl http://localhost:8000/health

# List served models
curl http://localhost:8000/v1/models | jq '.data[].id'
# → "dolphin-2.9.3-llama-3.1-8b"

# Quick chat test
curl -X POST http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"dolphin-2.9.3-llama-3.1-8b","messages":[{"role":"user","content":"ping"}]}'
```

## 4 — Key Environment Variables

### vLLM ROCm Variables

| Variable | Value | Purpose |
|----------|-------|---------|
| `HSA_OVERRIDE_GFX_VERSION` | `11.0.0` | Tells ROCm the exact gfx ISA for RDNA 3 — **required** |
| `HIP_VISIBLE_DEVICES` | `0` | Use only the first (primary) GPU |
| `VLLM_ATTENTION_BACKEND` | `ROCM_FLASH` | Enable ROCm Flash Attention |
| `VLLM_ROCM_USE_AITER` | `1` | Enable AITER optimisations for RDNA 3 |
| `HIP_FORCE_DEV_KERNARG` | `1` | Performance flag for gfx1100 |

### Brain Project Service Variables

| Variable | Service | Purpose |
|----------|---------|---------|
| `HUGGING_FACE_HUB_TOKEN` | vLLM | Download model weights from HuggingFace |
| `WEBUI_SECRET_KEY` | OpenWebUI | Session signing secret |
| `SEARXNG_SECRET` | SearXNG | Request signing secret |
| `JUPYTER_TOKEN` | Coding Agent | JupyterLab access token |
| `QDRANT_API_KEY` | OpenWebUI + Qdrant | Qdrant auth (optional) |

## 5 — Ollama Alternative (Simpler Setup)

If you prefer Ollama over vLLM (no HuggingFace token required):

```bash
docker compose -f node-a-vllm/docker-compose.ollama.yml up -d

# Pull a model
docker exec ollama_brain ollama pull llama3.1:8b
```

Ollama is served on **port 11435** to avoid conflicts.

### LiteLLM integration for Ollama on Node A

```yaml
  - model_name: brain-ollama
    litellm_params:
      model: ollama/llama3.1:8b
      api_base: http://192.168.1.9:11435
      timeout: 180
```

## 8 — Troubleshooting

### GPU not detected by ROCm

```bash
lsmod | grep amdgpu
ls -la /dev/kfd /dev/dri/render*
groups   # should include 'render' and 'video'
sudo modprobe amdgpu   # if /dev/kfd is missing
```

### brain-vllm container exits immediately

```bash
docker logs brain-vllm --tail 30
# Common causes:
# 1. /dev/kfd not found — ROCm not installed or kernel module not loaded
# 2. HSA_OVERRIDE_GFX_VERSION not set — already in compose, check env file
# 3. Out of VRAM — AWQ quantisation should keep usage under 12 GB
```

### vLLM health endpoint returns 503

The model may still be loading (can take 3–5 min on first start).

```bash
docker logs brain-vllm -f | grep -E "loading|loaded|error|Uvicorn"
```

### VRAM budget (RX 7900 XT — 20 GB)

| Component | Approximate VRAM |
|-----------|-----------------|
| dolphin-2.9.3-llama-3.1-8b AWQ | ~6–8 GB |
| Operating headroom | ~2 GB |
| Available for context | ~10 GB |

### Checking GPU utilisation

```bash
rocm-smi
watch -n 2 rocm-smi

# Via Hardware Agent API
curl http://192.168.1.9:8090/gpu
curl http://192.168.1.9:8090/status
```
